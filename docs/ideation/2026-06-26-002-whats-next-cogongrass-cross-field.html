What's next for cross-field cogongrass detection?

Ideation · 2026-06-26 · grounds on the completed arch_sweep results · extends the prior "best vision model" ideation. This is an idea menu, not a plan — pick one and run /ce-brainstorm on it.

Short answer: no, we haven't exhausted the space — and we can't yet even be sure what "best" is. The sweep so far explored exactly one family: a frozen foundation backbone + a small head + test-time adaptation (best = aimv2 + EATA, 0.839, tying the original ~0.84). Four whole families are untouched: (1) fixing the noisy ground truth we score against, (2) combining the models we already have (ensembles/calibration — nearly free off the cached features), (3) a different problem formulation (promptable detection / segmentation that re-anchors to each new field), and (4) remote-sensing-specific domain-generalization. The highest-leverage next move isn't a new backbone — it's the cheap combination of #1 and #2.

The ranked survivors

1Clean the test labels & fix the ruler medium · ceiling-raiser

What: Run the existing suspect_negatives.py / fp_audit.py to surface 0422 tiles the models confidently call cogongrass but ground truth marks negative, hand-correct them in label_tiles.py, and freeze a clean tiles_dataset_0422clean/ eval variant. Pair with evaluation hygiene: frame-level bootstrap confidence intervals + per-frame breakdown + report metrics against both noisy and clean ground truth.

Why it's #1: Labels are "not-boxed = negative" and "no box file = all-negative," so every missed annotation becomes a false negative in the answer key. AUROC ~0.90 with argmax recall ~0.6 is the fingerprint of label noise, not a weak model — the model already ranks these tiles correctly; the disagreement is the label. Until the ruler is clean (and has error bars — with only 2 collections, 0.804 vs 0.839 may be within noise), we literally cannot tell which architecture is best.

Impact: large (and compounds every other result) · Effort: medium (human-in-loop; tooling exists but needs repointing from the legacy 160/1280 config to 512/4096) · Harness: a VariantSpec over corrected labels + bootstrap_ci() in common.py; re-run the sweep, every number in the report becomes trustworthy.

Basis: our AUROC/recall gap; repo's own suspect_negatives.csv tooling; external SAM2-ELNet label-enhancement (arxiv 2503.12404).

2Decorrelated ensemble of the top backbones (+ per-member EATA) cheap · high-EV

What: Average the per-tile cogongrass probabilities of the top 4 backbones (siglip2, aimv2, cradio, dinov3_sat), each with its EATA-adapted head. Two tiers: average frozen heads, then average TTA-adapted heads.

Why: All four sit at AUROC ~0.90 but come from genuinely different pretraining objectives (image-text contrastive, autoregressive, agglomerative, satellite SSL), so their errors on a never-seen field are likely decorrelated even though their ranking quality is similar — the textbook setup where probability averaging jumps accuracy. EATA already beats baseline on each member individually (0.839/0.832/0.828/0.821).

Impact: +0.01–0.03 → plausibly ~0.85–0.86 · Effort: hours (pure cache reads + 4 tiny heads) · Harness: new models/ensemble.py loading cached features via features.load_features, reusing trainer.train_head + tta.adapt_head + _make_ok_row; one ResultRow with extra="ensemble=...".

Basis: our 10 cached feature banks at AUROC ~0.90; classic ensemble-of-decorrelated-learners.

3Use the operating threshold we already fit (calibration for the FN-costly setting) cheap · high-EV

What: The headline balanced accuracy is computed at argmax 0.5 — the F2-optimal threshold fit on 0606 (pick_threshold_on) is recorded but never applied. Apply it; add temperature scaling on 0606, AdaBN score alignment on 0422, and prior-matching (pick the 0422 threshold so predicted positive fraction matches known prevalence). Report cogongrass recall / F2 at that point.

Why: Argmax recall ~0.6 is the single largest source of false negatives — the costly error. The fitted op-thresholds (~0.30–0.44) are exactly what raises recall toward the ~0.85 regime AdaBN/RoTTA already reach. Prior-matching is label-free (uses only the unlabeled 0422 score distribution + the class prior), so it stays an honest cross-collection result.

Impact: cogongrass recall ~0.6 → ~0.85, large F2 gain · Effort: ~1 hour (scoring-path change in _make_ok_row) · Harness: add an eval_threshold policy; true conformal FN-rate control needs labels → route via the existing few_shot track.

Basis: trainer._make_ok_row argmax line; the unused pick_threshold_on / f2_sweep machinery.

4Deploy-time promptable detection with visual exemplars (T-Rex2 / T-Rex-Omni) new mechanism

What: Drop the frozen classifier. On a new field, supply 2–5 example boxes of cogongrass; an open-vocab/visual-prompt detector (T-Rex2, with T-Rex-Omni negative prompts for look-alike grasses) detects all similar regions, rasterized onto the same 512px tile grid → heatmap.

Why: It's the only approach that re-anchors per field at deploy time instead of freezing a decision boundary on 0606. Matching is relative ("find more like these, in this image"), which sidesteps the exact covariate shift (white balance, sun angle, phenology) TTA is clawing ~2 pts back against. Negative prompts directly target the costly false positives. Risk: cogongrass coverage is diffuse texture, not discrete countable objects, and the view is oblique — visual-prompt detectors may under-segment dense patches.

Impact: high upside on cross-field robustness; could exceed 0.84 because it adapts · Effort: medium (new standalone harness; runs on GB10) · Comparability: rasterize detections to the 0422 grid → identical balanced-accuracy / recall / AUROC protocol.

Basis: T-Rex2 (2403.14610), T-Rex-Omni negative prompts (2511.08997); plant-prompting calibration "Does Your VFM Speak Plant?" (2604.09920).

5The cheap cached-feature stack: CORAL alignment · feature-concat · multi-scale/flip TTA cheap · high-EV

What: Three label-free levers that all run off the cached features and stack with #2/#3: (a) CORAL — re-color 0606 features to 0422's second-order statistics before training the head; (b) early-fusion concat — concatenate 2–4 backbones' feature vectors into one wide head (learns cross-backbone interactions vs #2's late fusion); (c) multi-scale + flip TTA — cache tile256/tile512 + flipped views and average head probs.

Why: CORAL removes the dominant linear part of the covariate shift using only the unlabeled 0422 feature matrix (already cached) — complementary to AdaBN (which fixes the head's normalization, not its input). Concat captures complementary signal averaging can't. Multi-scale smooths nuisance scale/orientation variation.

Impact: +0.01–0.02 each, stackable · Effort: CORAL/concat hours (cache matmuls); multi-scale needs one fresh extraction pass per view · Harness: CORAL as an adaptation="coral" preprocessing in _train_frozen; concat in the ensemble combiner; views via data_variants.py + the per-(backbone,variant) cache.

Basis: cached features at AUROC ~0.90; CORAL/DANN domain adaptation; RS frequency-DG (FrogDogNet 2504.16433).

6SAM-family: tighten the labels & a segmentation/coverage reframe medium · ceiling + deliverable

What: Two uses of SAM2/SAM-3. (a) Label repair (offline): intersect each YOLO box with SAM masks so a tile is positive only where actual grass pixels (not the loose oblique rectangle's background corners) fall — replacing the crude ≥30%-of-box rule, lifting every track's ceiling. (b) Deploy reframe: "segment everything" → classify each mask region by texture / exemplar embedding → pixel-level coverage (the true deliverable, vs blocky tiles).

Why: Each 512px tile currently mixes cogongrass, other grass, soil and sky, so the label is applied to a heterogeneous patch — a tighter region is a cleaner, more transferable texture signal, and masks are the truest form of "coverage." sam_explore.py already exists (was capped at SAM2-tiny on the old 6 GB card; the GB10 unlocks SAM2-large / SAM 3).

Impact: high, partly compounding (label repair lifts the 0.817 baseline too) · Effort: low–medium (label repair is a one-time pass) · Comparability: collapse masks to per-tile labels for the standard metric; add pixel IoU / coverage-MAE as the richer native score.

Basis: SAM 3 concept/exemplar segmentation (2511.16719); ASAMPS prompt-free ag SAM2 (2026); repo's sam_explore.py.

7Make TTA deployment-real and compound it: per-frame SAR/CoTTA · fine-tune+TTA · ensemble-agreement self-training medium

What: A cluster that extends the winning TTA lever. (a) Per-frame robust TTA: add SAR and CoTTA to tta.py and an episodic per-frame mode — because heatmap_infer.py adapts one frame's tiles at a time (small, imbalanced batches where plain TENT collapses), unlike our whole-pool eval. (b) Stack fine-tune + TTA: run EATA on top of the LoRA-adapted backbone (currently the finetune path never calls adapt_head) — a clean A/B once LoRA finishes. (c) Ensemble-agreement pseudo-labeling: add 0422 tiles where ≥3/4 backbones agree at high confidence to training — agreement across decorrelated models is a far stronger reliability filter than single-model entropy (avoids confirmation bias).

Impact: (a) stabilizes the real heatmap; (b) +0.005–0.015 if shifts are complementary; (c) higher ceiling, higher variance · Effort: medium, mostly feature-space & CPU-testable like the rest of tta.py.

Basis: SAR / CoTTA / TestDG (2504.04981); our EATA result + the running LoRA cell; pseudo-labeling with ensemble gating.

The ceiling system (if you combined the survivors)

Cleanest high-confidence stack, mostly cheap: clean labels + error bars (#1) → score a CORAL-aligned (#5a) ensemble of EATA-adapted top-4 backbones (#2) at a prior-matched / F2 threshold (#3). That is achievable in roughly a day off the existing cache and plausibly lands ~0.85–0.87 on a trustworthy ruler. The genuinely new mechanism to test in parallel is visual-exemplar promptable detection (#4), the only path that adapts to each field rather than generalizing blindly — and the segmentation/coverage reframe (#6) is the one that matches the real deliverable.

Rejected — and why

IdeaWhy not
VLM as the classifierAlready falsified — Qwen2-VL-7B over all 7,006 0422 tiles scored ~0.572 balanced accuracy / 0.535 AUROC (near chance). Keep VLMs only as offline label-cleaners (#1), not detectors.
One-class / novelty detectionCogongrass is ~28% of the held-out tiles — not rare, so the "outlier" assumption breaks. Viable only as a re-ranking/abstention layer, not a standalone detector.
Bi-temporal change detectionBlocked by data: 0606 and 0422 are two different fields, not repeat flights of one. It's a "what to collect next" roadmap item (one revisit weeks apart unlocks it), not runnable today.
Naïve single-model self-trainingConfirmation bias amplifies the very label noise we're fighting. Only the ensemble-agreement-gated version (#7c) survives.
Yet another single backboneWe swept 10; AUROC is uniformly ~0.90. Adding an 11th frozen backbone is low-EV vs combining the ones we have (#2) or fixing the ruler (#1). (A weed-specific foundation model like WeedNet is a mild exception worth one slot, not a program.)

Grounding context

  • Status: frozen tournament (10 backbones, top 0.793) + TTA arm done — aimv2+EATA 0.839 beats the 0.817 baseline and ties the original ~0.84; LoRA/full fine-tune on aimv2 running now. Continued-SSL and few-shot harnesses built but not fully run.
  • Protocol: train collection 0606 → test entirely held-out field 0422 (7,006 tiles, ~28% positive); balanced accuracy + per-class recall; false negatives costly (F2). Single GB10 GPU; features cached per backbone.
  • Central finding: the ground truth is noisy ("not-boxed = negative"), so measured scores are a pessimistic floor and model ranking is uncertain without confidence intervals — which is why "fix the ruler" tops the list.
  • Axes covered: data/labels & evaluation · model combination & calibration · problem reframes (detection/segmentation/weak-supervision) · RS-specific domain generalization & robust TTA.
